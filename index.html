PostgreSQL Basics

Understanding Relational Databases

What is PostgreSQL?

PostgreSQL (or "Postgres") is a powerful, open-source object-relational database system with over 30 years of active development.

Key Concepts

Tables

Structured collections of related data organized in rows and columns

Primary Keys

Unique identifiers for each row in a table

Foreign Keys

Fields that reference primary keys in other tables

Indexes

Performance optimizations for faster queries

Why PostgreSQL for Our Quiz?

  • ✓ Reliability and data integrity
  • ✓ Complex query support
  • ✓ JSON support for flexible data
  • ✓ Scalability for growing applications

SQL Fundamentals

Structured Query Language Basics

What is SQL?

SQL (Structured Query Language) is the standard language for managing and querying data in relational databases like PostgreSQL.

Core SQL Operations

DML (Data Manipulation)

SELECT, INSERT, UPDATE, DELETE

DDL (Data Definition)

CREATE, ALTER, DROP

DCL (Data Control)

GRANT, REVOKE

TCL (Transaction Control)

COMMIT, ROLLBACK

Remember: SQL is declarative - you describe WHAT data you want, not HOW to get it

SQL CREATE Statement

Creating Database Tables

CREATE TABLE Syntax

CREATE TABLE table_name ( column1 data_type PRIMARY KEY, column2 data_type NOT NULL, column3 data_type DEFAULT default_value, ... );

Example: Creating a Users Table

CREATE TABLE users ( id SERIAL PRIMARY KEY, username VARCHAR(50) UNIQUE NOT NULL, email VARCHAR(100) NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );

CREATE INDEX Syntax

CREATE INDEX index_name ON table_name (column1, column2);
Tip: Indexes dramatically improve query performance on large tables

Table Operations

Adding and Removing Columns

Adding Columns with ALTER TABLE

-- Add a single column ALTER TABLE table_name ADD COLUMN column_name data_type; -- Add multiple columns ALTER TABLE table_name ADD COLUMN column1 data_type, ADD COLUMN column2 data_type; -- Add column with default value ALTER TABLE table_name ADD COLUMN column_name data_type DEFAULT default_value;

Modifying Columns with ALTER TABLE

-- Change column data type ALTER TABLE table_name ALTER COLUMN column_name new_data_type; -- Rename a column ALTER TABLE table_name RENAME COLUMN old_name TO new_name; -- Add NOT NULL constraint ALTER TABLE table_name ALTER COLUMN column_name SET NOT NULL;

Removing Columns with ALTER TABLE

-- Drop a single column ALTER TABLE table_name DROP COLUMN column_name; -- Drop multiple columns ALTER TABLE table_name DROP COLUMN column1, DROP COLUMN column2;

Other Table Operations

-- Rename a table ALTER TABLE old_name RENAME TO new_name; -- Drop a table completely DROP TABLE IF EXISTS table_name; -- Truncate a table (remove all data, keep structure) TRUNCATE TABLE table_name;
Warning: Always backup data before performing destructive operations!

PostgreSQL Data Types

Choosing the Right Type for Your Data

Text/String Types

VARCHAR(n) -- Variable-length string with limit TEXT -- Unlimited length string CHAR(n) -- Fixed-length string CHARACTER VARYING(n) -- Alternative to VARCHAR

Numeric Types

SMALLINT -- 2 bytes, -32,768 to 32,767 INTEGER -- 4 bytes, -2,147,483,648 to 2,147,483,647 BIGINT -- 8 bytes, large range integers DECIMAL(p,s) -- Exact decimal numbers NUMERIC(p,s) -- Alternative to DECIMAL REAL -- 4 bytes, single precision floating-point DOUBLE PRECISION -- 8 bytes, double precision floating-point SMALLSERIAL -- Auto-incrementing small integer SERIAL -- Auto-incrementing integer BIGSERIAL -- Auto-incrementing big integer

Date/Time Types

DATE -- Date only (year, month, day) TIME -- Time only (hour, minute, second) TIMESTAMP -- Date and time TIMESTAMPTZ -- Timestamp with time zone INTERVAL -- Time interval

Boolean & Special Types

BOOLEAN -- True or False values UUID -- Universally Unique Identifier JSON -- JSON data storage JSONB -- Binary JSON (indexed) ARRAY -- Array of values MONEY -- Currency amounts INET -- IP addresses MACADDR -- MAC addresses

Choosing the Right Type

For Names/IDs

Use VARCHAR with reasonable limits (50-100 chars)

For Descriptions

Use TEXT for long content, VARCHAR for short

For Numbers

Use INTEGER for counts, DECIMAL for money

For Dates

Use TIMESTAMPTZ for user input, TIMESTAMP for system

Tip: Use the most specific type that fits your data to optimize storage

SQL INSERT Statement

Adding Data to Tables

INSERT INTO Syntax

INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);

Example: Inserting a Single Row

INSERT INTO questions (question_text, correct_answer, option_a, option_b) VALUES ('What is the capital of France?', 'Paris', 'London', 'Berlin');

Example: Inserting Multiple Rows

INSERT INTO questions (question_text, correct_answer, option_a, option_b) VALUES ('What is the capital of France?', 'Paris', 'London', 'Berlin'), ('What is 2 + 2?', '4', '3', '5'), ('What is the capital of Spain?', 'Madrid', 'Barcelona', 'Valencia');

INSERT with SELECT (Copy Data)

Tip: String values must be enclosed in single quotes in SQL

SQL UPDATE & DELETE Statements

Modifying and Removing Data

UPDATE Syntax

UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;

Example: Updating a Question

UPDATE questions SET question_text = 'What is the capital of Spain?' WHERE id = 5;

DELETE Syntax

DELETE FROM table_name WHERE condition;

Example: Deleting a Question

DELETE FROM questions WHERE id = 3;

Advanced Examples

-- Update multiple rows UPDATE scores SET score = score * 1.1 WHERE player_name = 'Alice'; -- Delete multiple rows DELETE FROM scores WHERE score < 50; -- Delete all rows (dangerous!) DELETE FROM table_name;
Warning: Always use WHERE with UPDATE/DELETE unless you want to modify/delete ALL rows!

SQL ALTER Statement

Modifying Table Structure

ALTER TABLE Syntax

-- Add a new column ALTER TABLE table_name ADD COLUMN column_name data_type; -- Modify an existing column ALTER TABLE table_name ALTER COLUMN column_name data_type; -- Drop a column ALTER TABLE table_name DROP COLUMN column_name; -- Rename a table ALTER TABLE old_name RENAME TO new_name;

Example: Adding a Column

ALTER TABLE questions ADD COLUMN difficulty VARCHAR(10) DEFAULT 'medium';

Example: Changing a Column Type

ALTER TABLE scores ALTER COLUMN score INTEGER;

Example: Adding Constraints

-- Add NOT NULL constraint ALTER TABLE questions ALTER COLUMN question_text SET NOT NULL; -- Add UNIQUE constraint ALTER TABLE users ADD CONSTRAINT unique_email UNIQUE (email); -- Add CHECK constraint ALTER TABLE scores ADD CONSTRAINT valid_score CHECK (score >= 0);
Tip: Be careful with ALTER on production tables - it can lock the table

Row Level Security

Controlling Data Access

What is RLS?

Row Level Security (RLS) is a PostgreSQL feature that allows you to define policies that control access to rows in a table based on user attributes.

Enabling RLS

-- Enable RLS on a table ALTER TABLE table_name ENABLE ROW LEVEL SECURITY;

Creating Policies

-- Create a policy that allows users to see their own data CREATE POLICY "Users can view their own data" ON table_name FOR SELECT USING (user_id = auth.uid()); -- Create a policy that allows users to update their own data CREATE POLICY "Users can update their own data" ON table_name FOR UPDATE USING (user_id = auth.uid());

Policy Examples

-- Allow read access to authenticated users CREATE POLICY "Authenticated users can read" ON table_name FOR SELECT USING (auth.role= 'authenticated'); -- Allow all operations for specific users CREATE POLICY "Admins can do anything" ON table_name FOR ALL USING (auth.role = 'admin'); -- Allow operations based on data values CREATE POLICY "Users can edit their own posts" ON posts FOR UPDATE USING (author_id = auth.uid);

For Our Quiz Game

Since we're bypassing authentication, we'll use simple policies:

-- Allow all operations for everyone CREATE POLICY "Allow all operations" ON questions FOR ALL USING (true);
Important: RLS policies must be created after enabling RLS on table

Supabase Connection

Connecting Frontend to Database

Environment Variables

Supabase uses environment variables to securely connect your frontend to your backend database.

Required Environment Variables

# .env file SUPABASE_URL=your_supabase_project_url SUPABASE_ANON_KEY=your_supabase_anon_key

Finding Your Credentials

  1. Go to your Supabase project dashboard
  2. Navigate to Settings → API
  3. Copy Project URL and anon public key

JavaScript Client Setup

// Install Supabase JavaScript client npm install @supabase/supabase-js // Initialize client import { createClient } from '@supabase/supabase-js' const supabase = createClient( process.env.SUPABASE_URL, process.env.SUPABASE_ANON_KEY );
Security Note: The anon key allows public access but respects RLS policies

Supabase Data Operations

CRUD Operations with Supabase

Reading Data (SELECT)

// Fetch all questions const { data, error } = await supabase .from('questions') .select(*); // Fetch specific columns const { data } = await supabase .from('questions') .select('id', 'question_text', 'correct_answer');

Creating Data (INSERT)

// Insert a new question const { data, error } = await supabase .from('questions') .insert([ { question_text: 'What is the capital of Japan?', correct_answer: 'Tokyo', option_a: 'Seoul', option_b: 'Beijing' } ]);

Updating Data (UPDATE)

// Update a question const { data, error } = await supabase .from('questions') .update({ question_text: 'Updated question text?' }) .eq('id', 1);
Tip: Supabase methods return a Promise with { data, error } object
